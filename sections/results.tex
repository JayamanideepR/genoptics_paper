\section{Results}

This section reports the current returned CodeV/LLESP evidence for four questions: whether generic tabular generative models are usable optics baselines, whether the tuned optical representation improves over the untuned representation, whether synthetic expansion improves over patent-only training, and whether ray-tracing based post-processing improves generated prescriptions. Log-whitened material-parameterization results are excluded from this version of the paper.

Unless otherwise stated, each entry starts from a batch of 10,000 generated samples. We report the \emph{simulation-verified feasible sample fraction}: the percentage of generated samples that pass the post-processing and CodeV/LLESP preparation layer and then return from CodeV without an error. This denominator is stricter than a success rate on submitted samples because it also counts samples rejected before simulation.

\subsection{Benchmarking Tabular Generative Models}

Table~\ref{tab:benchmark_model_comparison} summarizes the current unconditional baseline results on the tuned representation. The benchmark set includes a SMOTE-style interpolation reference and three learned tabular generative baselines with returned CodeV outputs. The main observation is that the benchmark behavior is highly model-dependent: SMOTE-style interpolation remains easy to simulate, STaSy and TabSyn produce substantial numbers of usable designs on synthetic-expanded datasets, and TabDDPM produces almost no CodeV-successful prescriptions in the current runs.

\subsection{Effect Of Tuned Optical Representation}

The tuned representation is most important for learned generative models. Table~\ref{tab:tuned_untuned_results} compares tuned and untuned versions using matched dataset variants and the same 10,000-sample denominator. For TabSyn, the tuned representation improves the simulation-verified feasible sample fraction by 26.6 percentage points on the 250k synthetic-expanded dataset, 13.0 points on 500k, and 32.5 points on the full synthetic-expanded dataset. SMOTE is less sensitive to the representation, which is expected because it interpolates directly between existing samples rather than learning the same high-dimensional generative distribution.

\subsection{Effect Of Synthetic Dataset Scale}

The dataset-scale result should not be read as a monotonic scaling law. The current TabSyn runs show a clear separation between patent-only training and patent-plus-synthetic training, but the 250k, 500k, and full synthetic-expanded variants do not improve monotonically. This is still a useful result: synthetic expansion moves TabSyn from near-zero feasible yield on patent-only training to a 67--78\% simulation-verified feasible sample fraction in the tuned representation, while also revealing that model selection and training configuration matter at larger synthetic scales.

\subsection{Ray-Tracing Based Post-Processing Benefit}

The ray-tracing based post-processing layer is evaluated as an optional correction before CodeV/LLESP preparation. Table~\ref{tab:raytrace_postproc_benefit} shows that this step increases the end-to-end feasible yield for TabSyn tuned generations across all three synthetic-expanded dataset variants. The improvement is 11.0, 12.8, and 16.0 percentage points for 250k, 500k, and full synthetic-expanded training respectively.

The returned optical metric summaries show the same qualitative direction. Figure~\ref{fig:results_nrte_metric_delta_current} and Table~\ref{tab:raytrace_metric_delta} report paired common-row relative improvements at raw metric quantiles. For each percentile $p$, the reported value is $100$ times the no-NRTE raw metric $Q_p$ minus the NRTE raw metric $Q_p$, divided by the no-NRTE raw metric $Q_p$. Positive values indicate lower error-like metric values after ray-tracing post-processing. These quantiles are computed only on rows that were submitted in both branches and returned successfully in both CodeV runs.
