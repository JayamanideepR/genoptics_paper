\section{Dataset}

We construct a large-scale lens prescription dataset for generative modeling by combining a curated patent-derived reference set with a synthetic expansion generated from perturbations of those reference designs. Each row contains geometric prescription parameters, material information, design conditions, optical performance metrics, and provenance fields that allow generated benchmark datasets to be traced back to their source design family. The dataset is used in four variants: patent-only, patent plus 250k synthetic training samples, patent plus 500k synthetic training samples, and the full patent-plus-synthetic set. The 250k and 500k names refer to the number of synthetic training rows; all variants additionally include the fixed patent rows and the fixed synthetic validation/test rows.

\subsection{Patent-Derived Reference Designs}

The reference set contains 1,253 patent-derived optical system designs. For each design, we store the lens prescription in tabular form together with material labels, numerical material properties, operating conditions, and evaluated optical metrics. The corresponding CodeV sequence files are retained as reference source artifacts.

To keep the reference and synthetic datasets consistent, the current dataset assumes:
\begin{enumerate}
    \item Designed for object distance of infinity
    \item Realistic materials picked from existing catalogues
    \item Image and object space is of air material
    \item Image and object surfaces are considered to be flat/planar surfaces without any curvature
    \item Aspheric surfaces and grating are not considered
    \item Number of surfaces in the dataset range from 3 to 17
    \item A maximum field angle condition is considered as design specification
    \item Design performance is evaluated at [0,0.3,0.5,0.7,1.0] times the maximum field angle
    \item Field angles are represented in the meridional/y-field direction, with x-field angle entries set to zero
    \item Performance evaluated in visible spectrum using fixed wavelengths (F, d, C lines)
    \item Material between last surface and image plane is air and penultimate material is glass 
\end{enumerate}

Each optical system is represented as a sequence of surfaces with associated geometric and material parameters. Figure \ref{fig:lens_param} illustrates the lens assembly and the parameters used to define the design space. For each surface $j$, the dataset records:
\begin{enumerate}
    \item \textbf{Radius of curvature $R_j$}: The signed radius of curvature of surface $j$
    \item \textbf{Thickness $j$}: The axial distance between surfaces $j$ and $j+1$
    \item \textbf{Material $j$}: Label of Optical material immediately following surface $j$
    \item \textbf{Semi-diameter $j$}: Height of tip of surface $j$ from the optical axis
    \item \textbf{stop surface index}: Index of the surface designated as aperture stop
\end{enumerate}
The full released tables also contain object/image-surface fields, field and wavelength specifications, optical performance metrics, provenance columns, perturbation-lineage fields, filtering flags, and split labels; the complete column schema is provided in the appendix.

Along with design parameters, each optical system is associated with operating conditions that define its intended use:
\begin{enumerate}
    \item \textbf{Maximum field angle}: largest field angle to be accommodated by the system;
    \item \textbf{Effective focal length (EFL)}: target effective focal length;
    \item \textbf{Entrance pupil diameter (EPD)}: intended entrance pupil diameter;
    \item \textbf{Total length}: axial distance from the first surface to the image plane.
\end{enumerate}

To evaluate optical performance, each design is analyzed at field angles corresponding to $\{0, 0.3, 0.5, 0.7, 1.0\}$ times the maximum field angle. Some performance metrics are normalized by the diffraction-limited Airy disk radius to allow comparison across systems with different specifications:
\begin{equation}
r_{\mathrm{airy}} =  \frac{1.22\, \lambda_{\mathrm{ref}} \, \mathrm{EFL}}{\mathrm{EPD}}.
\end{equation}
The dataset records normalized spot-size metrics, encircled-energy metrics, and axial-field nominal performance metrics. Unless otherwise specified, distances are reported in millimeters, angles in degrees, and wavelengths in nanometers.

\subsection{Synthetic Expansion}

The synthetic dataset is generated by perturbing the patent-derived reference designs and re-evaluating the resulting candidate prescriptions. At a high level, the synthetic construction process is:
\begin{enumerate}
    \item sample a parent design from the patent-derived reference set;
    \item perturb curvature, thickness, material, or combinations of these quantities;
    \item re-evaluate semi-diameters and image-plane placement using the optical evaluation pipeline;
    \item discard rows that fail the generation or simulation checks;
    \item retain successful rows with full lineage back to the parent design and perturbation source.
\end{enumerate}

\subsection{Canonical Filtering And Splits}

All dataset variants are derived from a single canonical table. Synthetic rows preserve source folder, source row, parent patent family, perturbation lineage, and sequence-file provenance outside the model-facing feature matrix. The canonical filtering pass started from 1,126,448 rows and rejected 115 rows, all due to infinity-like tokens in tolerance-sensitivity columns. All other canonical filters in the current pass removed zero rows.

Splits are assigned at the parent-patent-family level so that a patent seed and its synthetic children do not cross train, validation, and test sets. The split procedure also enforces training coverage of categorical topology variables such as surface count, stop index, and active material/air indicators. The current audit reports zero leaking parent families for all dataset variants.
